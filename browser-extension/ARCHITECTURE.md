# 浏览器扩展架构设计

## 功能概述
在 GitHub/Gitee 仓库页面上添加"通过 GADA 一键部署"按钮，点击后唤起本地 GADA 应用并自动填入仓库 URL。

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                   浏览器扩展 (Chrome/Firefox)                 │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  1. 内容脚本 (content.js)                            │   │
│  │     - 检测 GitHub/Gitee 仓库页面                     │   │
│  │     - 注入"一键部署"按钮                             │   │
│  │     - 处理按钮点击事件                               │   │
│  └──────────────────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  2. 背景脚本 (background.js)                         │   │
│  │     - 与本地 HTTP 服务通信                           │   │
│  │     - 处理跨域请求                                   │   │
│  │     - 管理扩展状态                                   │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   本地 GADA HTTP 服务                        │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  1. HTTP 监听端点 (localhost:3000/api/browser)       │   │
│  │     - 接收浏览器扩展的部署请求                       │   │
│  │     - 验证请求格式和权限                             │   │
│  │     - 调用 GADA 部署逻辑                             │   │
│  └──────────────────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  2. WebSocket 服务 (可选)                            │   │
│  │     - 实时状态推送                                   │   │
│  │     - 部署进度反馈                                   │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

## 通信协议

### 1. 浏览器扩展 → 本地服务
```
POST http://localhost:3000/api/browser/deploy
Content-Type: application/json

{
  "repositoryUrl": "https://github.com/username/repo",
  "action": "deploy",
  "timestamp": "2024-01-01T12:00:00Z",
  "extensionId": "chrome-extension-xxx"
}
```

### 2. 本地服务响应
```
{
  "success": true,
  "message": "部署请求已接收",
  "deploymentId": "dep_123456",
  "statusUrl": "http://localhost:3000/api/deployments/dep_123456"
}
```

## 文件结构

```
browser-extension/
├── ARCHITECTURE.md          # 架构设计文档
├── README.md               # 扩展使用说明
├── shared/                 # 共享代码
│   ├── scripts/
│   │   ├── content.js      # 内容脚本（页面注入）
│   │   ├── background.js   # 背景脚本（扩展后台）
│   │   └── utils.js        # 工具函数
│   ├── styles/
│   │   └── inject.css      # 注入样式
│   └── locales/            # 多语言支持
│       ├── en.json
│       └── zh_CN.json
├── chrome/                 # Chrome扩展专用文件
│   ├── manifest.json       # Chrome扩展清单
│   └── icons/              # 图标文件
└── firefox/                # Firefox扩展专用文件
    ├── manifest.json       # Firefox扩展清单
    └── icons/              # 图标文件
```

## 实现步骤

### 阶段1：本地HTTP监听服务
1. 在现有GADA服务中添加浏览器扩展专用端点
2. 实现部署请求处理逻辑
3. 添加跨域支持（CORS）

### 阶段2：浏览器扩展开发
1. 创建Chrome扩展基础结构
2. 实现内容脚本注入
3. 添加GitHub/Gitee页面检测
4. 实现"一键部署"按钮

### 阶段3：通信集成
1. 实现扩展与本地服务的HTTP通信
2. 添加错误处理和重试机制
3. 实现状态反馈和进度显示

### 阶段4：测试和优化
1. 跨浏览器测试（Chrome/Firefox）
2. 本地服务连通性测试
3. 用户体验优化

## 安全考虑

1. **来源验证**：只接受来自localhost的请求
2. **输入验证**：严格验证仓库URL格式
3. **权限控制**：扩展仅请求必要的权限
4. **错误处理**：优雅处理各种异常情况

## 兼容性要求

- Chrome 88+ (支持 Manifest V3)
- Firefox 89+
- GADA 服务版本 >= 1.0.0
- Node.js >= 18.0.0
