## 第二章：项目部署全流程

### 2.1 准备工作

#### 检查系统环境
```bash
# 运行环境检查
node --version    # 需要≥18.0.0
npm --version     # 需要≥6.0.0
git --version     # 可选但推荐
```

#### 确保服务器运行
```bash
# 启动Web服务器
npm run ui

# 验证运行
curl http://localhost:3000/api/health
# 应该返回: {"status":"ok"}
```

### 2.2 部署流程详解

#### 步骤1：添加项目
1. 点击 **"添加项目"** 按钮
2. 输入GitHub仓库URL
3. 点击 **"分析仓库"**

#### 步骤2：AI分析阶段
**系统自动分析**：
1. **仓库信息**：名称、描述、星标数
2. **项目类型**：React、Vue、Node.js等
3. **技术栈**：使用的框架和库
4. **构建工具**：webpack、vite等
5. **依赖管理**：package.json、requirements.txt等

**分析报告示例**：
```
📊 分析报告
项目: vuejs/vue
类型: Vue.js前端框架
技术栈: Vue 3, Vite, TypeScript
构建命令: npm run build
启动命令: npm run dev
建议端口: 3000
依赖: Node.js ≥16, npm/yarn/pnpm
```

#### 步骤3：部署前预检
**自动检查**：
```
🔍 预检清单:
1. ✅ 端口3000可用
2. ✅ 磁盘空间充足 (需要500MB)
3. ✅ Node.js已安装 (v18.17.0)
4. ✅ Git已安装 (v2.39.0)
5. ✅ 网络连接正常
6. ✅ 系统权限足够
```

**如果有问题**：
```
❌ 发现问题:
1. 端口3000被其他程序占用
2. Node.js版本太低 (当前v16.0.0)

🛠️ 建议解决方案:
1. 点击"一键修复"自动更换端口
2. 按照指南升级Node.js
```

#### 步骤4：自动部署
**部署进度**：
```
🚀 开始部署...
[ 10%] 克隆仓库... ✓
[ 30%] 安装依赖... ⏳ (可能需要几分钟)
[ 60%] 配置项目... ✓
[ 80%] 构建项目... ⏳
[100%] 启动服务... ✓

🎉 部署完成!
访问地址: http://localhost:3000
管理地址: http://localhost:3000/admin
```

#### 步骤5：验证部署
```bash
# 方法1：浏览器访问
打开 http://localhost:3000

# 方法2：API检查
curl http://localhost:3000/api/health

# 方法3：查看日志
在Web界面点击"查看日志"
```

### 3.1 启动项目

1. 选择项目
2. 点击 **🟢 启动** 按钮
3. 等待启动完成
4. 查看状态变为"运行中"

#### 启动参数配置
```
启动配置:
- 端口: 自定义端口号
- 环境: development/production
- 内存限制: 512MB
- CPU限制: 1核心
```

### 3.2 停止项目

#### 正常停止
1. 选择运行中的项目
2. 点击 **🔴 停止** 按钮
3. 确认停止操作
4. 等待停止完成

#### 停止方式
```
停止选项:
- 优雅停止: 发送SIGTERM信号
- 强制停止: 发送SIGKILL信号
- 超时设置: 30秒后强制停止
```

#### 停止日志示例
```
[INFO] 开始停止项目: my-app
```

#### 重启场景
- 修改配置后生效
- 内存泄漏需要重启
- 更新代码后重启
- 定期维护重启

#### 重启流程
```
重启步骤:
1. 停止项目
2. 清理缓存
3. 重新构建（如果需要）
4. 启动项目
```

```
📦 备份包含:
1. 项目源代码

3. 配置文件 (.env, config files)

```

#### 备份选项
```
备份设置:
- 备份位置: 本地/远程/云存储
- 备份频率: 每日/每周/每月
- 保留策略: 保留最近7个备份
- 压缩方式: zip/tar.gz
- 加密选项: AES-256加密
```

#### 手动备份
```
手动备份步骤:
1. 选择项目
2. 点击"备份项目"
3. 选择备份位置
4. 设置备份选项
5. 开始备份
6. 等待完成
```

#### 自动备份
```
自动备份配置:
- 启用: ☑️ 启用自动备份
- 时间: 每天02:00
- 位置: /backups/projects
- 保留: 最近30天
- 通知: 备份完成后邮件通知
```
