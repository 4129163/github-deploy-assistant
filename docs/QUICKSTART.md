## 📋 前提条件

确保你已经：
- ✅ 安装了Node.js（版本≥18.0.0）
- ✅ 安装了GitHub Deploy Assistant
- ✅ 知道如何打开终端/命令提示符

**如果不确定，先运行这个检查**：
```bash

```

✅ **应该看到类似**：
```
v18.17.0
9.6.7
```

---

## 🚀 步骤1：启动Web服务器

### 1.1 打开终端

- **Windows**：按 `Win + R`，输入 `cmd`，回车
- **macOS**：打开"应用程序"→"实用工具"→"终端"
- **Linux**：按 `Ctrl + Alt + T`

### 1.2 进入项目目录

```bash
# 如果你安装在用户目录
cd ~/github-deploy-assistant

# 或者如果你安装在别的地方
cd /你的/安装/路径/github-deploy-assistant
```

### 1.3 启动服务器

```bash
npm run ui
```

✅ **应该看到**：
```
GitHub Deploy Assistant Web UI
服务器运行在: http://localhost:3000
按 Ctrl+C 停止服务器
```

❌ **如果看到错误**：
- `端口被占用` → 运行 `PORT=4000 npm run ui`，然后访问 `http://localhost:4000`
- `模块找不到` → 运行 `npm install` 先安装依赖

### 1.4 保持终端打开

**重要！** 不要关闭这个终端窗口，否则服务器会停止。

http://localhost:3000

## 🎯 步骤3：部署第一个项目

### 3.1 点击"添加项目"按钮

**位置**：左上角，一个绿色的 "+" 按钮

**点击后**：会弹出对话框让你输入GitHub链接

### 3.2 输入示例项目链接

在输入框中粘贴：
```
https://github.com/facebook/create-react-app
```

这是一个官方的React示例项目，非常适合测试。

### 3.3 点击"分析仓库"按钮

**等待**：AI会分析这个仓库，大约10-30秒

✅ **分析完成后你会看到**：

**项目信息卡片**：
```
项目名称: create-react-app
项目类型: React应用
描述: 官方React脚手架工具
建议端口: 3000
依赖: Node.js, npm
```

**AI分析报告**：
```
这是一个React前端项目，使用webpack构建。
启动命令: npm start
构建命令: npm run build
测试命令: npm test
```

### 3.4 点击"一键部署"按钮

**开始自动部署**，你会看到：

```
部署进度: 0% → 100%
步骤:
1. 克隆仓库... ✓
2. 安装依赖... ✓
3. 配置项目... ✓
4. 启动服务... ✓
```

⏱️ **等待时间**：根据网络速度，2-5分钟

✅ **部署完成后**：
- 项目状态变为"运行中"
- 显示访问地址：`http://localhost:3000`
- 显示"停止"按钮（绿色变成红色）

---

## 🔍 步骤4：验证项目运行

### 4.1 在浏览器访问项目

打开新标签页，访问：
```
http://localhost:3000
```

✅ **应该看到**：
```
React的蓝色旋转图标
"Edit src/App.js and save to reload."
```

🎉 **恭喜！** 你的第一个项目已经成功部署并运行！

### 4.2 查看实时日志

在Web界面中：
1. 点击项目列表中的 `create-react-app`
2. 点击右侧的 **"查看日志"** 按钮

✅ **会看到**：
```
[日志] 项目已启动
[日志] 编译完成
[日志] 监听端口3000
```

### 4.3 查看系统监控

点击右上角的 **"监控"** 按钮，可以看到：
- CPU使用率
- 内存使用情况
- 磁盘剩余空间

## ⚙️ 步骤5：项目管理操作

### 5.1 停止项目

点击右侧的 **红色"停止"按钮**

✅ **看到**：
- 项目状态变为"已停止"
- 按钮变成绿色"启动"
- 浏览器访问 `http://localhost:3000` 会显示无法连接

### 5.2 启动项目

点击 **绿色"启动"按钮**

✅ **看到**：
- 项目状态变回"运行中"
- 可以再次访问

### 5.3 重启项目

点击 **蓝色"重启"按钮**

**用途**：当修改配置后需要重新加载

### 5.4 删除项目

**注意**：删除后无法恢复！

1. 点击 **"删除项目"** 按钮
2. 确认对话框点击"确定"

✅ **看到**：
- 项目从列表中消失
- 文件从磁盘删除

---

## 🧪 步骤6：运行项目诊断

### 6.1 点击"项目诊断"按钮

位置：右侧工具栏，橙色图标

### 6.2 等待诊断完成

系统会检查：
```
✅ 端口是否可用
✅ 依赖是否完整
✅ 配置文件是否正确
✅ 磁盘空间是否充足
```

### 6.3 查看诊断报告

✅ **健康项目会显示**：
```
项目状态: 健康
问题: 无
建议: 无
```

❌ **有问题会显示**：
```
项目状态: 异常
问题:
1. 端口3000被占用
2. Node.js版本太低
建议:
1. 点击"一键修复"自动解决
2. 或按照指南手动修复
```

### 6.4 使用"一键修复"

如果有问题，点击 **"一键修复"** 按钮

✅ **系统会自动**：
- 更换被占用的端口
- 安装缺失的依赖
- 修复配置错误

---

## 💾 步骤7：备份和恢复项目

### 7.1 创建备份

点击 **"备份项目"** 按钮

选择备份位置：
- 本地磁盘
- 云存储（如果配置）

✅ **备份完成后**：
- 显示备份文件路径
- 显示备份时间

### 7.2 恢复项目

如果需要恢复：
1. 点击 **"恢复项目"** 按钮
2. 选择备份文件
3. 确认恢复

✅ **恢复完成后**：
- 项目回到备份时的状态
- 所有配置和数据都恢复

## ❓ 常见问题

### Q1: 部署一直卡在"安装依赖"怎么办？
**A**: 
1. 点击"取消"按钮
2. 检查网络连接
3. 点击"项目诊断"查看问题
4. 使用"一键修复"解决网络问题

### Q2: 端口3000被占用怎么办？
**A**:
1. 点击"修改端口"按钮
2. 输入新端口（如4000）
3. 点击"保存"
4. 重新部署

### Q3: 如何查看详细的错误信息？
**A**:
1. 点击"查看日志"按钮
2. 查看红色错误信息
3. 或按F12打开浏览器开发者工具查看控制台

### Q4: 部署成功后如何更新项目？
**A**:
1. 启用Webhook自动更新
2. 或手动点击"更新代码"按钮
3. 然后点击"重新构建"
