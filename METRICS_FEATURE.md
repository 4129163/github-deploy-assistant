# 资源监控仪表盘功能文档

## 功能概述

为GitHub Deploy Assistant新增了完整的资源监控仪表盘功能，包括：
- **实时监控**: CPU、内存、磁盘使用率
- **历史曲线**: 使用ECharts展示历史数据趋势
- **进程树**: 显示系统进程树和资源占用
- **端口占用**: 查看系统端口使用情况
- **拓扑图**: 可视化的系统资源拓扑

## 新增文件

### 后端文件
1. `src/routes/metrics.js` - 资源监控API路由
   - `/api/metrics/current` - 获取当前系统指标
   - `/api/metrics/history` - 获取历史监控数据
   - `/api/metrics/process-tree` - 获取进程树信息
   - `/api/metrics/ports` - 获取端口占用信息
   - `/api/metrics/summary` - 获取监控摘要
   - `/api/metrics/project/:projectId` - 获取项目资源使用情况

### 前端文件
1. `public/metrics.html` - 资源监控仪表盘页面
2. 主页面(`index.html`)新增监控仪表盘链接

### 配置文件更新
1. `src/server/index.js` - 注册metrics路由到Express应用

## 功能特性

### 1. 实时监控指标
- **CPU监控**: 总使用率、各核心详情、系统负载
- **内存监控**: 使用率、总量、已用、可用
- **磁盘监控**: 各分区使用情况、总量、已用空间
- **网络监控**: 网络接口状态、连接信息

### 2. 历史数据图表
- **CPU历史曲线**: 显示CPU使用率变化趋势
- **内存历史曲线**: 显示内存使用率变化趋势
- **系统负载曲线**: 显示1/5/15分钟负载变化
- **磁盘分布图**: 饼图显示各分区使用比例

### 3. 进程树管理
- **进程层级**: 显示父子进程关系
- **资源占用**: 实时显示CPU和内存占用率
- **进程详情**: PID、用户、命令、参数
- **资源分类**: 高/中/低资源占用标记

### 4. 端口占用监控
- **端口列表**: 显示所有占用端口
- **协议状态**: TCP/UDP、监听/连接状态
- **程序关联**: 关联端口与运行程序
- **端口描述**: 常见端口的功能描述

### 5. 项目资源监控
- **项目进程**: 监控GADA管理的项目进程
- **资源占用**: 项目CPU/内存使用情况
- **端口监听**: 项目监听的端口
- **运行状态**: 项目运行时长和状态

## 前端功能

### 监控仪表盘页面
访问地址: `http://localhost:3000/metrics.html`

**主要组件**:
1. **概览卡片**: CPU、内存、磁盘、进程的实时状态
2. **历史图表**: 使用ECharts绘制的历史趋势图
3. **进程表格**: 显示Top 20进程及其资源占用
4. **端口表格**: 显示系统端口占用情况
5. **时间范围选择**: 5分钟、15分钟、1小时、6小时
6. **自动刷新**: 每30秒自动更新数据

### 主页面集成
在主页面(`index.html`)的侧边栏系统信息部分，新增了"查看详细监控"按钮，点击可跳转到监控仪表盘。

## 技术实现

### 后端实现
- **数据采样**: 每5秒采样一次系统指标
- **历史存储**: 内存中存储最多1小时的历史数据
- **进程信息**: 使用`ps`命令获取进程树
- **端口信息**: 使用`ss`或`netstat`命令获取端口占用
- **安全执行**: 使用`safeExec`包装系统命令执行

### 前端实现
- **图表库**: 使用ECharts 5.4.3进行数据可视化
- **响应式设计**: 支持桌面和移动设备
- **实时更新**: WebSocket或定时轮询获取最新数据
- **数据缓存**: 客户端缓存历史数据，减少API请求

### 数据流
```
系统指标采样 → 内存存储 → API端点 → 前端请求 → ECharts渲染
    ↓
历史数据缓存   实时数据推送   用户交互
```

## 性能考虑

1. **内存使用**: 历史数据最多保存720条（1小时数据），每条约1KB，总计约720KB
2. **CPU开销**: 采样间隔5秒，系统命令执行有超时限制和错误处理
3. **网络传输**: 数据经过压缩，API响应经过优化
4. **前端性能**: ECharts图表使用虚拟渲染，大数据量时性能良好

## 安全考虑

1. **命令执行**: 所有系统命令通过`safeExec`执行，有超时限制
2. **数据过滤**: 进程和端口信息经过过滤，避免敏感信息泄露
3. **输入验证**: API参数经过验证，防止注入攻击
4. **权限控制**: 遵循系统现有权限控制机制

## 使用说明

### 启动监控
1. 启动GADA服务: `npm start`
2. 访问主页面: `http://localhost:3000/`
3. 点击侧边栏的"查看详细监控"按钮
4. 或直接访问: `http://localhost:3000/metrics.html`

### 监控功能
1. **实时监控**: 页面自动刷新显示最新数据
2. **历史查看**: 选择时间范围查看历史趋势
3. **进程管理**: 查看进程资源占用，识别资源消耗大的进程
4. **端口检查**: 检查端口占用，排查端口冲突

### 故障排查
1. **数据不更新**: 检查API连接状态，刷新页面
2. **图表不显示**: 检查ECharts库加载，检查网络连接
3. **进程信息缺失**: 检查系统权限，可能需要root权限
4. **端口信息不全**: 系统可能限制非root用户查看所有端口
