# GitHub Deploy Assistant - 可视化Web界面

## 概述

这是一个完全按钮化的GitHub Deploy Assistant Web界面，彻底消灭了命令行输入框，所有操作都通过直观的按钮完成。界面采用现代设计，支持响应式布局，可在桌面、平板和手机上使用。

## 主要特性

### 🎯 完全按钮化操作
- **所有操作都通过按钮完成**，无需输入命令
- **直观的按钮分组**：核心操作、配置管理、高级操作、诊断工具
- **智能按钮状态**：根据项目状态自动启用/禁用相关按钮

### 🎨 现代化UI设计
- **响应式布局**：适配各种屏幕尺寸
- **深色/浅色主题**：支持一键切换
- **实时状态显示**：系统状态、项目状态一目了然
- **流畅动画效果**：按钮点击、页面切换都有平滑动画

### 🔧 功能分组

#### 1. 核心操作组（绿色按钮）
- 启动项目
- 停止项目
- 重启项目
- 查看日志

#### 2. 配置管理组（蓝色按钮）
- 修改端口
- 环境变量
- 更新依赖
- 重新构建

#### 3. 高级操作组（紫色按钮）
- 备份项目
- 回滚项目
- 项目诊断
- 删除项目

#### 4. 诊断工具组（橙色按钮）
- 运行诊断
- 健康检查
- 性能监控
- 一键修复

#### 5. 部署向导组（大号按钮）
- 一键智能部署
- 分步向导部署
- 远程服务器部署

### 📊 实时监控
- **系统状态面板**：CPU、内存、磁盘使用情况
- **项目状态监控**：实时显示项目运行状态
- **活动时间线**：记录所有操作历史
- **实时日志**：WebSocket实时推送日志

## 快速开始

### 安装依赖
```bash
cd /path/to/github-deploy-assistant
npm install
```

### 启动Web界面
```bash
# 启动Web服务器（端口3000）
npm run ui

# 或使用开发模式（自动重启）
npm run ui:dev
```

### 访问界面
打开浏览器访问：http://localhost:3000

## 使用指南

### 1. 部署新项目
1. 点击左侧"添加项目"按钮
2. 输入GitHub仓库URL
3. 点击"分析仓库"按钮
4. 点击"一键部署"按钮

### 2. 管理现有项目
1. 在左侧项目列表中选择项目
2. 使用右侧的按钮组进行管理：
   - 启动/停止/重启项目
   - 修改配置
   - 运行诊断
   - 备份/回滚

### 3. 系统诊断
1. 点击"运行诊断"按钮
2. 查看诊断报告
3. 使用"一键修复"解决发现的问题

## API集成

### 前端API调用
所有按钮操作都通过REST API与后端通信：

```javascript
// 项目操作API
POST /api/projects/{id}/start     # 启动项目
POST /api/projects/{id}/stop      # 停止项目
POST /api/projects/{id}/restart   # 重启项目

// 部署API
POST /api/deploy                  # 部署项目
POST /api/repo/analyze           # 分析仓库

// 诊断API
POST /api/projects/{id}/diagnose  # 运行诊断
POST /api/projects/{id}/fix       # 修复问题
```

### WebSocket实时通信
- 实时日志推送
- 项目状态更新
- 系统状态监控

## 开发说明

### 项目结构
```
public/
├── index.html          # 主页面
├── css/
│   └── style.css      # 样式文件
├── js/
│   ├── main.js        # 主逻辑
│   ├── ui.js          # UI交互
│   ├── api.js         # API通信
│   └── diagnose.js    # 诊断模块
└── assets/            # 静态资源

server.js              # 后端API服务器
package.json           # 项目配置
```

### 技术栈
- **前端**：纯HTML/CSS/JavaScript，无框架依赖
- **后端**：Node.js + Express
- **通信**：REST API + WebSocket
- **样式**：CSS Grid + Flexbox
- **图标**：Font Awesome 6

## 故障排除

### 常见问题

#### 1. 无法启动服务器
```bash
# 检查端口占用
lsof -i :3000

# 或使用其他端口
PORT=4000 npm run ui
```

#### 2. API连接失败
- 检查后端服务器是否运行
- 检查API地址配置
- 查看浏览器控制台错误

#### 3. 按钮无响应
- 检查网络连接
- 刷新页面
- 清除浏览器缓存

#### 4. 样式显示异常
- 检查CSS文件加载
- 清除浏览器缓存
- 使用开发者工具检查

### 调试模式
在浏览器控制台可以启用调试模式：
```javascript
localStorage.setItem('debug', 'true')
```
