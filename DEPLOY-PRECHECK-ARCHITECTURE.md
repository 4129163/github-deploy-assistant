# 部署前预检清单功能架构设计

## 功能概述
在部署项目前，自动检测系统环境和项目依赖，提供详细的检测报告和一键修复功能。

## 检测项清单

### 1. 端口占用检测
- **检测内容**：检查项目指定的端口是否被占用
- **检测方法**：使用 `netstat` 或 `ss` 命令
- **修复方案**：自动查找可用端口或停止占用进程

### 2. 磁盘空间检测
- **检测内容**：检查目标部署目录的可用空间
- **阈值**：至少需要项目大小 * 2 的空间
- **修复方案**：清理临时文件或提示更换目录

### 3. 依赖检测
- **Git**：检查是否安装，版本是否满足要求
- **Node.js**：检查版本，npm/yarn 是否可用
- **Python**：检查版本，pip 是否可用
- **Docker**：检查是否安装，Docker Compose 是否可用
- **Java**：检查 JDK 版本
- **其他**：根据项目类型检测特定依赖

### 4. 系统库检测
- **Linux**：检查 libc、glibc、openssl 等
- **Windows**：检查 Visual C++ Redistributable
- **macOS**：检查 Xcode Command Line Tools

### 5. 权限检测
- **目标目录**：检查是否有读写权限
- **系统命令**：检查是否有执行权限
- **端口绑定**：检查是否有绑定特权端口的权限

### 6. 环境变量检测
- **PATH**：检查必要命令是否在 PATH 中
- **项目相关**：检查必要的环境变量是否设置

## 模块设计

### 后端模块
1. **DeployPrecheck** 主类
   - 协调所有检测器
   - 生成检测报告
   - 管理修复流程

2. **检测器基类** (BaseDetector)
   - 统一的检测接口
   - 结果格式化
   - 优先级排序

3. **修复器基类** (BaseFixer)
   - 自动修复接口
   - 修复步骤记录
   - 回滚机制

### 前端组件
1. **PrecheckWizard** 组件
   - 分步检测向导
   - 实时结果显示
   - 一键修复按钮

2. **IssueCard** 组件
   - 问题展示卡片
   - 修复状态指示
   - 手动修复指南

3. **ReportView** 组件
   - 检测报告展示
   - 导出功能
   - 历史记录查看

## 数据流设计

```
用户请求预检 → 路由层 (/api/precheck) → PrecheckService
    ↓
PrecheckService 调用检测器 → 生成检测报告
    ↓
前端展示检测结果 → 用户选择一键修复
    ↓
PrecheckService 调用修复器 → 执行修复 → 返回修复结果
```

## API 设计

### 1. 启动预检
```
POST /api/precheck/start
{
  "projectId": "string",
  "checkItems": ["ports", "disk", "dependencies", "permissions"]
}
```

### 2. 获取预检结果
```
GET /api/precheck/result/:checkId
```

### 3. 一键修复
```
POST /api/precheck/fix/:checkId
{
  "issueIds": ["string"] // 可选，不传则修复所有
}
```

### 4. 获取修复进度
```
GET /api/precheck/fix-progress/:fixId
```

## 与现有系统集成

### 2. 集成到部署流程

- 如检测失败可中断部署
- 提供修复建议

### 3. 前端集成
- 在部署向导中添加"预检"步骤
- 在项目详情页添加"运行预检"按钮
- 在仪表板显示预检状态

## 安全考虑

1. **权限控制**
   - 修复操作需要用户确认
   - 系统级操作需要管理员权限
   - 记录所有修复操作日志

2. **隔离性**
   - 修复操作在沙箱中执行
   - 不影响系统其他部分
   - 提供回滚机制

3. **验证机制**
   - 修复后重新检测验证
   - 用户确认修复结果
   - 失败时提供详细错误信息
